十分钟搞定自己的Github博客

第一步:创建Github 域名和空间

免费个人博客搭建， 这是一篇小白也能看懂的文章，本文主要针对OS X ，Windows 除了软件安装方式和命令有些区别（装了git bash也一样），其他基本一样。
你可能经常会看到类似于这样的博客：

我们需要先在github上搭建一个远程仓库,这个仓库就是以后别人访问我们博客的地方

1. 创建Github 域名和空间

1.1注册

首先你需要注册一个Github账号，已有的可以下翻去1.2 创建仓库，注意github的username，这会影响到你的域名，你的域名将会是 username.github.io ，所以认真的取个名字吧。

注册过程可能需要验证你的邮箱，其他就不在赘述。

1.2 创建github仓库

在你的github中创建一个仓库(repository)来存储我们的博客,点击右上角你的github图像,就会有new repository按钮创建仓库,然后出现如下画面,输入仓库名称,名称需要注意,比如我的设置的是xincheng123.github.io,其中xincheng123是我github的用户名称,你的也必须(注意必须)是你的github用户名称,然后后面和我写的一样就可以了,然后点击Create repository 按钮创建即可

第一步就已经完成了，下面是创建本地厂库,以后我们写好微博就放在我们本地的一个仓库里,然后上传到github上,就可以让别人通过网络访问我们的博客了。

第二步 创建本地仓库

1 安装依赖程序列表

Hexo 可以说是目前最流行的博客框架了，基于Nodejs，更多信息可以google，下面需要安装的工具包括 :

  • HomeBrew
  • Git
  • nvm
  • Nodejs&&npm
  • 最终安装Hexo。
2.1安装HomeBrew，如果已安装HomeBrew 无需执行下面这行代码
/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"
2.2 验证HomeBrew是否安装成功,查询版本
brew  -v
3.1 安装git
brew install git
3.2 验证是否安装成功,查询版本
git  --version
4.1安装nvm
brew  install  nvm
4.2 安装完成后，CMD + Q 来重启终端
5.1安装Node.js和npm（默认安装Node.js就已经安装了npm）
brew   install  node
5.2 检查是否安装成功Node.js和npm
node  -v

npm  -v
6.1安装hexo
npm  install  -g  hexo-cli

这一步安装过程可能有点长,耐心等待

6.2检查安装hexo是否成功
hexo  -v

以上如果都安装成功,那么接下来就可以搭建本地仓库了

7 在本地创建一个厂库,也就是创建一个文件夹

  1. 在你喜欢的地方创建一个文件夹,文件夹名称任意起,最好用英文(这里比如你创建爱你的文件夹名称是hexo)

  2. 在终端cd到你创建的厂库目录文件夹hexo（这个文件夹名称任意)

    cd  /Users/zxc/hexo
8.创建hexo仓库
hexo init

这一步也需要一会,耐心等待

9.安装hexo仓库依赖包
nmp install    （这个命令好像不用敲，我敲了也没事）

到这里,你的本地仓库也创建完毕,本地创库创建完成,默认会在你的博客中生成一篇helloworld的博客,而且默认会有一个普通的博客主题样式(个人觉得很难看,后面会告诉搭建如何优化个人博客主题)

#####10 现在你可以先看看本地仓库中的个人博客页面是如何的

######10.1.把本地仓库中的博客生成静态网页，并开启hexo服务

  1. cd到hexo目录下(我这里是hexo目录下,你的也就是通过终端进入你刚刚自己创建的文件夹下,此时这个文件夹下应该已经有很多文件,暂时不要管)执行下面的命令生成静态页面

    hexo generate  （hexo g也可以）

  2. 启动本地服务，进行文章预览调试，注意执行这个命令后就不要动终端了,也不要退出终端命令：

    hexo server

  3. 浏览器输入http://localhost:4000就可以看到我们自己本地博客的静态网页了

自此本地厂库创建完成,现在你可以在终端中同时按键盘的 ctrl + C 来关闭本地服务

第三步 把本地创库部署到github创库上去

1.在本地的hexo目录下找到_config.yml文件，在这个文件中配置一些信息
2. 基础配置：打开_config.yml文件修改几个键值对，下面把几个必须设置的列出来按需求修改，记得 cmd + s 保存。先只修改红框中的数据

  1. repo后面那个网址的获取方法:

3. 安装hexo-deployer-git自动部署发布工具 ,在终端输入:
npm install hexo-deployer-git --save
4. cd到hexo目录下,执行下面命令生成静态网页(只要对hexo中文件进行了修改最好就执行一下下面这个命令)
hexo g
5. 把本地的博客部署到github上去(同上一个命令一样,都需要在hexo目录下)
hexo d

这一步也需要一段时间,耐心等待.期间可能会要你按任意键继续,你按下任意键(比如回车键)就继续了,然后如果这是你的第一次，终端会让你输入Github 的邮箱和密码，正确输入后，骚等片刻，就会把你的博客上传至Github 了。以后在每次把博客写完后，执行一下这个命令就可以直接发布了.

恭喜你能走到这一步，你的博客已经完成了，在浏览器中输入

https：//xincheng123.github.io

就能够访问了。xincheng123是我github的用户名,这里替换成你的用户名即可.(直接通过xincheng123.github.io访问也可以).你的可能主题和我的不一样,稍后告诉你怎么修改:
