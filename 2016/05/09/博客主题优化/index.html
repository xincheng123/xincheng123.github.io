博客主题优化

第一步:从官网下载主题或者直接clong主题到下面的文件夹中

官方主题下载地点

王魏大神的博客主题

王魏博客主题

1.clong主题

从github上clong下来你喜欢的博客主题,然后把Clong下来的文件复制到下图中的vno文件夹下(vno文件夹是我自己创建的,文件夹名称后面需要用到),这个文件夹名称用于设置给第2步的theme字段

2.最后修改hexo/_config.yml文件中的theme字段为vno
3.修改文件后,需要生成静态网页,然后开启hexo服务,先本地查看效果

hexo g //生成静态网页

hexo server //开启hexo服务

打开localhost:4000查看效果了。

4.如果效果满意,就可以部署到github上了

hexo g

hexo d

注意,所有的命令必须cd 到hexo文件夹下,到此为止,你的个人博客部署和优化已经完成,以后一些细节配置,主要就是修改_config.yml这个配置文件

5.所有的博客源文件都保存在hexo目录下的下面这个目录下

6.hexo 删除博客

删除仓库里面 source/_posts/我的文章.md这种带.md结尾的文件即可删除博客

7.执行下面命令更新博客

hexo g == hexo generate#生成
hexo s == hexo server #启动服务预览
hexo d == hexo deploy#部署

8.编写博客可以使用这个软件来编写，当然这只是为了更方便博客内容的排版而已

9.使用Mou编写博客需要了解makeDown语法

hexo n “name”命令来新建一个页面
scaffolds目录下，是新建页面的模板，可以在这里根据你自己的需求添加一些默认值

打开.md文章，markdown编辑，用上面三步发布

command + b : 加粗文字

添加图片：

  1. 放图片： 如在文章 “基数排序 (Radix Sort).md”的同一个目录下建一新文件夹，名字与文章名同名，把图片放进去（如果 _config.yml 的 post_asset_folder 已设为 true，则每次“hexo new post 文章名”时都会自动创建这样的一个文件夹）

  2. 在markdown编辑文章时引用图片，格式为：

    {% asset_img 图片名（带后缀） 图片描述%}

在此时虽然看不到，但部署到网页后就可看到。

title: postName #文章页面上的显示名称，可以任意修改，不会出现在URL中

date: 2013-12-02 15:30:16 #文章生成时间，一般不改，当然也可以任意修改

category: example #分类

tags: [tag1,tag2,tag3] #文章标签，可空，多标签请用格式，注意:后面有个空格

description: 附加一段文章摘要，字数最好在140字以内。
